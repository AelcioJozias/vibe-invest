# Rules

1. An investment cannot exist without being linked to an account.
2. In the first month of usage, the user only provides initial investment values and contributions.
3. In the first month, there is not enough historical data to calculate metrics such as average interest, cash return, or comparative growth.
4. In the first month, visualization should consider only contribution amounts and the total invested amount available at that moment.
5. From the following month onward, especially on the first day of each month, the user should update the current values of each investment.
6. Accepted movement types for monthly investment evolution are: `INVESTMENT_CREATED`, `CONTRIBUTION`, `INTEREST`, and `ADJUSTMENT`.
7. `INVESTMENT_CREATED` represents the initial posting of an investment.
8. `CONTRIBUTION` represents a money addition made by the user to the investment.
9. `INTEREST` represents returns generated by the investment.
10. `ADJUSTMENT` represents a manual value correction to fix inconsistencies, such as a typo or an exceptional user-reported adjustment.
11. Monthly return calculation must always compare the current value with the consolidated value from the previous month.
12. Monthly cash gain must be measured based on the difference between the previous month total and the current total, excluding inflows classified as contributions.
13. The interest calculation reference must never use the current month in isolation without considering the previous month base value.
14. Contributions are not part of the interest gain calculation.
15. Only movements classified as `INTEREST` must compose the interest return calculation.
16. Corrections must not be automatically treated as return; they are operational adjustments and must remain separated from interest measurement.

## Measurement Example

1. If in July the total invested amount was `10.000` and in August it became `11.000`, there is a gross variation of `1.000`.
2. This variation cannot be fully classified as return without separating what came from `CONTRIBUTION`, `INTEREST`, and `ADJUSTMENT`.
3. To measure real return, the backend must consider only postings of type `INTEREST` within the period.

## Dashboard Impact

1. When there is no previous month for comparison, comparative metrics must be empty, zeroed, or follow an explicit rule defined by the application.
2. The total monthly contribution amount must sum only `CONTRIBUTION` movements.
3. The total monthly interest gain must sum only `INTEREST` movements.
4. Net worth growth may consider consolidated net worth variation in the period, but interest return must remain separate from contributions.
